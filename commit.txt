Fix: carry-forward readings lost from primitive int fields — 2026-06-29

domain_streams.py DomainStreamAggregator.refresh:
  When a stream is not due (TTL not expired), its cached readings are carried
  forward into sig.readings. But the 12 int primitive fields (sig.kinetics etc.)
  start at 0 on every fresh DomainSignal and are only updated by _set() calls
  inside due-stream functions. Carried-forward readings were in sig.readings but
  never replayed into int fields — kinetics, recognition, fidelity (and others fed
  by slow streams) silently dropped to 0 on every non-forced sweep.
  Fix: after the stream loop, replay all readings (carried + fresh) into the int
  fields via max(current, r.alert). Due-stream fields are already set and the max
  is idempotent; carried fields get their correct alert level restored.

Fix Pacific arc routing: split arcs at date line — 2026-06-29

Root cause: Leaflet 1.9.4 normalises out-of-range longitudes (e.g. lon=-225)
back to their [-180,+180] equivalent (+135) when rendering, so arcs sent to the
Pacific-copy frame appeared on the Eastern-Hemisphere side and routed eastward
over the Atlantic/Africa/India.

solar_seismic.html:
  adjLon(): simplified back to standard [-180,+180] normalisation (Leaflet's
    own range — no more out-of-range shifts).
  arcPoints() → arcSegments(): computes 24-point westward arc in a continuous
    frame (lons may go below -180), normalises each point to [-180,+180], then
    SPLITS into separate polyline segments at each date-line crossing (jump >90°).
    Returns array of point arrays; render() now iterates and adds each segment.
    Each segment stays within [-180,+180] so Leaflet renders it without wrapping.

Fix Pacific arc routing: interpolated westward arc, robust adjLon — 2026-06-29

solar_seismic.html:
  adjLon(): replaced `lon > 0 ? lon-360 : lon` with modulo normalization to
    [-340, +20] — the full Pacific-centric range — handles all edge cases.
  arcPoints(): replaced 3-point tent with 12-point interpolated arc.
    Explicitly forces westward routing (`if a2 > a1: a2 -= 360`) before
    interpolating, so arcs always route over the Pacific regardless of whether
    Leaflet normalizes the endpoint coordinate. Arc apex uses sin(πt)*28 for
    smooth bow. All 12 intermediate points are in the adjLon frame (guaranteed
    negative), eliminating any possibility of Leaflet normalizing them back to
    positive Eastern-Hemisphere coordinates.

current sweep's due_names have their cached readings appended to sig verbatim.
