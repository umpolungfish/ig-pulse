## Stream data source fixes (7 of 7)

1. deribit_options — "insufficient data"
   Root cause: API no longer returns option_type field.
   Fix: parse from instrument_name suffix (-P = put, -C = call).

2. yield_curve — "no data"
   Root cause: Treasury fiscaldata filter query has spaces in URL (URL encoding
   issue) AND the security_desc values ("Treasury Notes 2-year") don't exist in
   the API — only "Treasury Notes" as one combined category.
   Fix: FRED T10Y2Y CSV (fred.stlouisfed.org) — returns 10Y−2Y spread directly.
   Note: FRED requires plain urllib (no custom User-Agent); _text() blocked.

3. vix — "no data"
   Root cause: CBOE cdn.cboe.com/api/... returns HTTP 403.
   Fix: Yahoo Finance chart API for ^VIX with Mozilla User-Agent.

4. shipping — "no data"
   Root cause: VesselFinder API returns HTML (requires paid API key).
   Fix: Yahoo Finance BDRY ETF (Breakwave Dry Bulk Shipping) — 5-day close
   series; day-over-day % change → Γ granularity / Σ stoichiometry signal.

5. power_grid — "no data"
   Root cause: EIA API v2 requires registration; api_key=public returns 403.
   Fix: CAISO OASIS (oasis.caiso.com) — free, no key, ZIP CSV of RTM 5-min
   load forecast. Sums MW across all 37 reporting areas → Ç kinetics signal.

6. gdelt — "no data"
   Root cause: GDELT doc API (api.gdeltproject.org/api/v2/doc/) rate-limits
   aggressively (HTTP 429 on every real collect).
   Fix: GDELT v2 raw 15-min event export via data.gdeltproject.org/gdeltv2/
   lastupdate.txt → ZIP CSV. 740 events per 15-min window, columns 26 (Goldstein
   scale) and 30 (AvgTone) used for coupling/granularity signals.

7. stereo_cr — "magnetic-only (no recent particle data)"
   Root cause: During quiet heliosphere, STEREO IMPACT particle channels are
   null — this is expected, not an error. Stream was still producing valid
   magnetic helicity readings (winding, chirality) and appending to errors.
   Fix: Removed the erroneous sig.errors.append(); stream now silently uses
   magnetic data when particle data is absent.
