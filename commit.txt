## geo_viz.py: SPACE_SOURCE_MAP

Added SPACE_SOURCE_MAP dict (origin["source"] → SPACE_ORIGINS key) so space-based
stream readings render at their actual orbital positions instead of falling back to
NOAA SWPC Boulder CO via PROJECTION_MAP.

Mapping:
  "sun"                → SPACE_ORIGINS["sun"]              (lon=0, lat=0)
  "L1_lagrange"        → SPACE_ORIGINS["L1_dscovr"]        (lon=-65, lat=5)
  "DSCOVR"             → SPACE_ORIGINS["L1_dscovr"]        (lon=-65, lat=5)
  "ACE"                → SPACE_ORIGINS["L1_ace"]           (lon=-75, lat=-5)
  "GOES-18" / "GOES-19"→ SPACE_ORIGINS["goes_18"]          (lon=-137, lat=0)
  "stereo_a"           → SPACE_ORIGINS["stereo_a"]         (lon=70, lat=0)
  "stereo_sept"        → SPACE_ORIGINS["stereo_a"]         (lon=70, lat=0)
  "earth_magnetosphere"→ SPACE_ORIGINS["earth_magnetosphere"](lat=90, lon=0)

Updated _extract_geo_nodes() priority chain:
  1. origin["lat"] / origin["lon"] if present  (seismic/atmosphere/tide — exact GPS)
  2. SPACE_SOURCE_MAP via origin["source"]       (ACE/GOES/STEREO/DSCOVR/SEPT/Sun/Kp)
  3. PROJECTION_MAP hub                          (market/social/bio/infrastructure)

Result: 25 of 73 nodes now render as origin_type="space" at correct orbital coords.
